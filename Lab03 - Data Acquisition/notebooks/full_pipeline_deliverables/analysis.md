📊 Book Market Intelligence Analysis
Executive Summary

This project collected book-related data from three different sources: a local library database, the GitHub API, and the Books to Scrape website. The goal was to gather information about books, borrowing behavior, and online technology trends related to books. The collected data was stored in a centralized SQLite database and analyzed to identify patterns in genres, ratings, borrowing frequency, and online repository activity.

Data Collection Statistics

The system collected data from three sources:

Database: Library records containing book information and borrowing history.

API: GitHub repositories related to book or publishing technologies.

Web Scraping: Product information from the Books to Scrape website.

All collected data was stored in structured tables and logged for traceability.

Market Insights
Popular Genres

The analysis shows that some genres contain more books than others, indicating higher representation in the library database.

Rating Trends

Average ratings vary between genres, which may indicate reader preference for certain types of books.

Borrowing Patterns

Borrowing frequency shows that some books are borrowed much more often than others, suggesting higher demand for specific titles.

Web Data Insights

Scraped data provides information about book availability and pricing from an online bookstore, which can help understand market pricing trends.

Technology Trends

GitHub API data highlights repositories related to books and publishing tools, which can indicate technological development trends in the publishing ecosystem.
